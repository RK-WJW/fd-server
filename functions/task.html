Task

task功能比较复杂，首先从界面开始介绍。

Task简介

界面方面可以设置一个task的本地目录，放置所有开发者自己编写的task文件，编写完毕后可以适用FDS来载入界面统一执行。

可能有的小伙伴说完全可以使用nodejs直接在终端执行，但是如果FDS安装在了开发远程机，那么通过这个task模块，我们就可以无需登录，通过界面进行shell操作。

而且task文件内置了开发者可以以task形式来同时操作本地和远程机脚本的能力。

载入后即可选择执行对应task任务，并有实时终端反馈。

Task模块配置说明

Task模块用于实现自动化任务处理，通过配置自动帮你执行一系列本地和远程的命令，执行处理逻辑主要由sheller完成。

sheller

配置方式

在你指定的目录下创建配置脚本（js）, 可以创建多个，配置好了之后，在页面点击reload，会将你在目录下创建的所有任务列出来，然后点击相应任务Run执行即可，下方会实时显示命令执行的日志信息。

配置脚本示例

//Task 配置文件格式 示例

/* code */

module.exports = {
   options : {
        //内置属性 设置本地命令执行的路径环境
        localWorkPath : "E://workspace/",
        //远程机 名称自定义 
        "rs001" : {
            host : "100.100.100.100",//ip
            username : "test001",//用户名
            password : "123456", //密码
            workPath : "/fd-server/test"//远程命令工作路径
        },
        "rs002" : {
            host : "100.100.100.100",
            username : "test002",
            password : "123456"
        },
        //自定义参数
        svnPath: "https://svn...."
    },
    test: {
        options: {
            //将覆盖上面设置的值
            localWorkPath : "E://workspace/test1/",
            //自定义参数
            name: "test"
        },
        task: [
            {
                //options中的参数可以通过[%= name %]方式取到
                "command": "echo [%= name %]"
            },{
                "id": "test1",//可选
                "command": "echo test1"
            },{
                "command": "echo test2",
                //after 可选 命令成功执行后对其结果进行再处理
                "after": function (data){
                    return {
                        data: data.replace(/(^\s+)|(\s+$)/, ''),
                        status: 1
                    };
                }
            },{
                //命令支持function形式
                "command": function (prev){
                    console.log(prev.ret.status == 1);//true
                    console.log(this.getResult("test1"));//test1
                    console.log(this.getResult());//test
                    console.log(this.getResult(2));//test2
                    console.log(this.options.name);//test options
                    console.log(this.task[0].command);//echo test                    
                    console.log(this.task[0].ret);//test
                    return "skip";//继续下一条命令
                }
            },{
                command: "echo test remote",
                //可选 执行一个远程机，命令将在指定的远程机上执行
                remote: "rs001"
            },{
                command: "exit"//终止向下执行
            },{
                command: "echo not exec"//将不会执行
            }
        ]
    },
    test2: {
        ... ...
    }
};

options

任务参数配置

  • localWorkPath 指定本地命令执行的路径

  • remote 远程机信息配置，属性名自定义，值必须满足一下格式：

    {
  host : "",//ip
  username : "",//用户名
  password : "", //密码
  workPath : ""//远程命令工作路径 可选
}
  • 其他 根据需要自定义一些参数

task

命令流程，数组的形式将任务流程的一系列命令配置出来。 每一条命令的配置是一个对象（object）形式，具体如下：

{
  id:"",
  command: "",
  remote: "",
  after: Function
}
  • id 命令的唯一标识 可选

  • command 命令, 支持String和Functoin形式

    • String直接指定命令语句，可以使用[%= paramName %]方式获取options中的参数
    • Function 通过函数处理，返回一个命令语句
    function (prev){
  //prev 参数 是前一条命令对象，命令结果存放在ret属性值中
  //执行函数的this对象
  /*
  {
    task: Array 当前任务
    options: Object 执行当前任务的配置参数
    getResult: Function 获取执行命令的结果,可通过命令在任务中的索引或者id属性指定，默认0
    prev:前一个命令对象
  }
  */
  return "exit";
}

  • remote 指定命令执行的远程机，这里只指定属性名称，具体远程机信息需要在options中配置 可选，不指定则是在本地执行

  • after 对结果在加工的处理函数 可选

    function (result){
  //result参数 是当前命令执行的结果字符串
  //执行函数的this对象 同上
  return result;
}

其他：exit和skip为内置命令, exit为终止命令继续向下执行, skip为跳过当前命令继续向下执行.
