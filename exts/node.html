.node

基础用法

有的时候我们往往需要模拟一些后端接口，或者做一些动态解析的功能，或者扩展动态路由。fd-server支持给vhost的静态服务扩展路由。 只需要在静态目录下放置 .node 的扩展名文件，代码书写如下：

//someroute.node
route(function(req,res){
   res.writeHead(200,{'Content-Type':'text/plain'});
   res.end('someroute');
});

那么当你访问对应host域名下得/someroute.node地址则会自动转入当前文件的route函数路由中，可随意扩展。

内置方法

在.node文件中编写的代码语法是nodejs的语法，但是由于是动态执行，所以为了解决路径问题和终端log文件，我给.node扩展了几个内置方法。

1.addModule 加载当前路径下node_modules里的模块,以.node文件所在目录为准.

2.require 加载node的原生模块。

3.logger 可打印日志到日志文件(用法同log4js，是log4js的一个实例)

4.route 路由入口函数

具体的.node实现机理为:

vm.runInNewContext(code, {
    logger: logger,
    Buffer: Buffer,
    '__dirname': dirname,
    addModule: function(mod) {
        return require(dirname + '/node_modules/' + mod);
    },
    require: require,
    route: function(run) {
        run(req, res);
    }
});

code为FDS拿到的.node文件源码字符。

https://github.com/SinaBlogFE/blogcombine 为一个实际的例子，是现在新浪博客的实时打包路由的代码.
